รู้อยู่ที่ไหน

ออกไปกี่ใบ กลับมากี่ใบ เหลือในคลังเท่าไหร่ — เห็นทั้งวงจรในหน้าเดียว ตั้งแต่ผ่านประตูออก จนกลับเข้าชั้นวาง

วิธีอ่านภาพจำลองนี้ — ทุกอย่างมาจากข้อมูลจริง
  • ช่องมีของ ในชั้นวาง = ติดขอบเขียว + มีกล่องอยู่ — คำนวณจากตำแหน่งที่มีกล่องจริงในระบบ
  • ช่องว่าง = ถาดสีเข้ม ไม่มีขอบเขียว — สัดส่วนมีของ/ว่างเปลี่ยนตามอัตราครองพื้นที่จริง
  • รถ INBOUND / DELIVERY แสดงอยู่ตลอด — จะ สว่างเต็ม เมื่อมีการยิง Gate เข้า/ออกจริงในวันนี้ ถ้ายังไม่มีรายการวันนี้ รถจะแสดงแบบ จางลง (ยังไม่ทำงาน)
  • กล่อง/หุ่นยนต์ที่เคลื่อนที่ วิ่งตามลูกศรแสดงทิศทางการไหลของงาน (cross-dock) ตลอดเวลา — บอกลำดับ เข้า → เก็บ → หยิบ → ออก
  • จุดสถานะแต่ละโซน — จุดเขียวเรืองแสง = โซนนั้นมีของ/กำลังทำงาน (พร้อมตัวเลขจริง เช่น มีของ, รอเก็บ, ส่งวันนี้), จุดเทา = ว่าง/ยังไม่มีงาน
  • กองพัก Putaway (ซ้าย) แสดงเมื่อมีกล่องผ่าน Gate เข้า รอเก็บเข้าชั้นจริง — กองขาออก Outbound (ขวา) แสดงตามจำนวนที่ส่งออก วันนี้ (กล่องที่ผ่าน Gate ออกแล้วจะไม่ track ต่อ จึงไม่นับกล่องที่ออกไปวันก่อน)
